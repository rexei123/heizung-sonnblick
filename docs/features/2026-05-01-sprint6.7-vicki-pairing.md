## Codec-Iterationen (Sprint 6.8)

### Iteration 1 — Sprint-5-Foundation-Codec (PR #14)
Sprint-5-Codec war fuer Test-Mock-Frames mit fPort=1, Command 0x01. Echter Vicki sendet aber fPort=2, Command 0x81 (Periodic Reporting v2). **Mismatch -> kein Decoding.**

### Iteration 2 — Offizieller MClimate-GitHub-Decoder (PR #38)
[`mclimate/lorawan-devices/vendor/mclimate/vicki.js`](https://github.com/mclimate/lorawan-devices/blob/master/vendor/mclimate/vicki.js) uebernommen. Lokal mit Node erfolgreich getestet (Output: korrekte Werte). **Aber im ChirpStack-Goja silent fail.**

### Iteration 3 — Strict-mode-Bug erkannt (PR #40)
ChirpStack-Goja-Engine laeuft im **strict-mode**. Offizieller MClimate-Code hat globale Variablen ohne `var`-Deklaration:

```js
toBool = function(value) { return value == '1' };  // implicit global!
tmp = ("0" + bytes[6].toString(16)).substr(-2);    // implicit global!
motorRange1 = tmp[1];                              // implicit global!
```

In strict-mode -> `ReferenceError: toBool is not defined`. ChirpStack swallowt den Fehler und sendet das Event ohne `object`-Feld.

**Fix:** Minimale Eigen-Implementierung (~70 Zeilen, alle `var`-deklariert) fokussiert auf Periodic Reporting v1/v2. snake_case-Aliase (`temperature`, `target_temperature`, `battery_voltage`, `motor_position`, `motor_range`) am `return`-Statement, damit der FastAPI-Subscriber unveraendert bleibt. camelCase-Felder fuer ChirpStack-UI-Anzeige bleiben auch.

Nach Codec-Replace + `docker compose restart chirpstack` (Cache-Reload) greift der Decoder. Verifiziert mit Vicki-001-Frame `gRRsTaiZAeAw` -> Setpoint 20°C matches Display.

## Lessons Learned

1. **Foundation-Codecs aus Mock-Tests sind keine Production-Codecs.** Vor dem ersten Pairing offiziellen Decoder vom Hersteller einbinden.
2. **ChirpStack-Goja ist strict-mode.** Codecs ohne `var` brechen silent. Eigene Codecs immer mit `'use strict'` lokal testen, bevor in ChirpStack-UI uebernehmen.
3. **ChirpStack-Cache:** Codec-Update im UI ist nicht sofort live, ChirpStack-Container braucht Restart (oder die Application-Cache laeuft ab nach X Min).
4. **Browser-Submit verifizieren:** „Device profile updated"-Toast = persistiert. Ohne Toast: nochmal probieren.
5. **MQTT-Auth-Topic-Scoping:** heizung-api darf nur `application/+/device/+/event/...` lesen. Ein Diagnose-Subscribe auf `eu868/gateway/+/event/+` mit heizung-api-User wird abgelehnt — chirpstack-User dafuer nutzen.
