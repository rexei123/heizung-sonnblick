### D2 Inline-Edit-Pattern

Wiederverwendbares Muster: `LabelCell` in
`frontend/src/app/devices/page.tsx:208-306` (Sprint 9.13a TA3,
HF-9.13a-1).

Beschreibung:
- `useState(editing|draft|error)` lokal pro Zelle
- Klick auf Edit-Icon (Material-Symbol `edit`) öffnet Input mit
  `autoFocus`, `autoComplete="off"`
- `onKeyDown`: Enter → speichern (TanStack-Query-Mutation), Esc →
  abbrechen (Draft zurücksetzen)
- `onBlur` → ebenfalls speichern
- Loading-Indicator `progress_activity` Spin-Icon während `mutateAsync`
- Fehler-Anzeige als `role="alert"` unter dem Input
- Idempotenz: wenn neuer Wert identisch zum aktuellen, kein API-Call

Pattern ist UI-spezifisch auf eine Komponente zugeschnitten; für eine
Liste von Werten in `/einstellungen/temperaturen-zeiten` müsste das
Muster in eine generische Komponente extrahiert werden — oder pro
Wert dupliziert eingesetzt.

Es gibt kein zweites Inline-Edit-Muster im Repo
(`/raumtypen/page.tsx` nutzt full-page Edit-Form über Mode-State,
nicht inline).
